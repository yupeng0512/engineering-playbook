# 投研报告生成 Skill（Investment Research Report Generator）

## Description

投研报告生成专家。当用户需要为项目生成面向 VC/投资人的完整市场调研报告、商业计划书（BP）、融资路演材料时，使用此 Skill。

适用关键词：投研报告、商业计划书、BP、融资路演、市场调研、投资分析。

---

## 工作流程

### Step 1：收集输入

启动时需确认以下信息（缺失项主动询问）：

1. **项目名称**
2. **项目简介**（一段话描述：做什么、给谁用、解决什么问题）
3. **目标用户**
4. **当前阶段**（概念期 / MVP / 已上线 / 已有收入）
5. **融资目标**（金额 + 轮次）
6. **团队信息**（可选）
7. **已有数据**（用户数、收入等，可选）

### Step 2：市场调研

- 使用 Web Search 搜索最新行业数据
- **所有数据必须标注来源**（艾媒咨询、民政部、国家统计局等权威机构）
- 竞品数据至少两个来源交叉验证
- 财务预测必须标注"假设"，不可将预测伪装成事实

### Step 3：生成 10 模块报告

### Step 4：自检 Review

## 报告结构（10 大模块）

### 模块 1：执行摘要（Executive Summary）
- **一句话定位**：做什么、给谁用、解决什么问题
- **核心亮点**：3-5 个 bullet point（市场机会、差异化优势、商业模式、团队背景）
- **融资需求**：本轮金额、出让股比、资金用途概览
- **关键里程碑**：已达成 + 未来 12-18 个月目标

### 模块 2：市场分析（Market Analysis）
- **宏观背景**：政策、人口、社会趋势等驱动因素
- **TAM**（Total Addressable Market）：整个行业总市场规模
- **SAM**（Serviceable Available Market）：产品可触达的市场范围
- **SOM**（Serviceable Obtainable Market）：短期可获取的市场份额
- **增长驱动力**：3-5 个推动市场增长的核心因子
- 数据来源必须可查证

### 模块 3：行业痛点与机会（Problem & Opportunity）
- **用户痛点清单**：至少 3 个核心痛点，每个配数据佐证
- **现有解决方案的不足**：竞品为什么没解决好
- **市场空白/机会窗口**：为什么"现在"是最佳切入时机

### 模块 4：产品与解决方案（Product & Solution）
- **产品形态**：平台类型、技术栈、交付方式
- **核心功能矩阵**：功能名称 → 解决的痛点 → 用户价值
- **产品路线图**：Phase 1/2/3 含时间线
- **技术壁垒/护城河**：算法、数据、网络效应、品牌等

### 模块 5：商业模式（Business Model）
- **收入模型**：每条收入线的名称、定价策略、预估 ARPU、付费转化率
- **单位经济模型**：CAC、LTV、LTV/CAC（≥3）、回收期
- **收入增长飞轮**：闭环增长逻辑图

### 模块 6：竞品分析（Competitive Analysis）
- **竞品矩阵表**：目标用户、核心功能、商业模式、融资阶段、用户规模、差异化
- **竞争优势总结**：2-3 句话说明"我们为什么能赢"

### 模块 7：运营与增长策略（Go-to-Market Strategy）
- **冷启动策略**：首批用户从哪来
- **增长渠道**：按优先级排序，标注预估 CAC
- **留存策略**：核心留存手段
- **病毒系数/裂变机制**：K-factor 设计
- **关键运营指标**：DAU/MAU/留存/付费率目标值

### 模块 8：财务预测（Financial Projections）
- **3-5 年收入预测表**：注册用户、MAU、付费用户、付费率、ARPU、总收入、运营成本、净利润
- **乐观/基准/保守三场景对比**
- **关键假设说明**：每个数字背后的假设逻辑必须透明
- **盈亏平衡分析**

### 模块 9：团队与融资（Team & Funding）
- **核心团队**：创始人 + 核心成员背景
- **融资历史 + 本轮计划**：金额、估值逻辑、资金用途拆分、下一轮触发条件

### 模块 10：风险分析与应对（Risk Analysis）
- **风险矩阵**：至少 5 个风险（市场/竞争/政策/技术/运营）
- 每个风险含：描述、概率、影响程度、应对策略

## 数据真实性要求

1. **所有市场数据必须标注来源**
2. **优先使用权威数据源**：
   - 政府部门：民政部、国家统计局、工信部
   - 研究机构：艾媒咨询、艾瑞咨询、QuestMobile、易观
   - 行业报告：36氪、投中网、清科研究中心
3. **财务预测必须标注"假设"**
4. **竞品数据应交叉验证**：至少两个来源印证

## 输出格式规范

### 文档格式
- Markdown 格式，标题层级 ≤ 4 级
- 关键数据用 **加粗** 突出
- 表格对齐，数字右对齐
- 每个模块末尾加分割线

### 语言风格
- **面向投资人的专业语调**：简洁、数据驱动、逻辑严密
- 避免过度营销语言，用数据说话
- 中英文术语并用（TAM、ARPU、LTV/CAC）
- 关键结论前置（金字塔原理）

### PPT 大纲（可选输出）
如需同时输出 PPT 大纲，按以下结构：
1. 封面（项目名 + 一句话定位 + 融资信息）
2. 痛点（3 张 slide，每张一个核心痛点 + 数据）
3. 解决方案（产品截图 + 核心功能）
4. 市场机会（TAM/SAM/SOM 漏斗图）
5. 商业模式（收入模型图）
6. 竞品对比（矩阵表）
7. 增长策略（增长飞轮图）
8. 财务预测（3年收入曲线图）
9. 团队介绍
10. 融资计划（资金用途饼图）
11. 愿景（结束页 + 联系方式）

---

## Review 检查清单

完成报告后，必须逐项检查：

- [ ] 所有数据标注了来源
- [ ] TAM → SAM → SOM 逻辑自洽（SAM < TAM，SOM < SAM）
- [ ] 收入模型的假设合理且保守
- [ ] LTV/CAC ≥ 3（如不满足需解释改善路径）
- [ ] 竞品分析客观，不回避自身劣势
- [ ] 财务预测有乐观/基准/保守三个场景
- [ ] 风险分析全面，应对策略具体可执行
- [ ] 语言专业、简洁，无夸大其词
- [ ] 整体叙事逻辑流畅：问题 → 机会 → 方案 → 可行性 → 回报
