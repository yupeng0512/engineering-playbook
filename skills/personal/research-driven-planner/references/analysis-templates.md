# 分析模板参考

> 本文档定义 Deep Dive 和 Pattern Extraction 阶段使用的分析模板。

---

## 方案深度分析卡片

对 Top 3 候选方案，各填写一张分析卡片。

```markdown
# 方案分析：[名称]

## 基本信息
- **链接**：[URL]
- **Stars / 用户量**：
- **许可证**：
- **技术栈**：
- **维护状态**：活跃 / 维护中 / 衰退

## 它如何解决核心问题

用 2-3 段话描述：
1. 它的整体架构是什么样的？
2. 它用什么方式解决了我们面临的核心挑战？
3. 它的数据流 / 用户流是怎样的？

## 架构决策分析

| 决策点 | 它的选择 | 理由（推测） | 对我们的启示 |
|--------|---------|-------------|-------------|
| 例：数据存储 | PostgreSQL | 需要关系查询 + JSONB 灵活性 | 与我们的栈一致 |
| 例：任务调度 | Celery + Redis | 异步任务 + 重试 | 可考虑复用 |
| ... | ... | ... | ... |

## 可复用的设计模式

1. **模式名称**：简述这个模式做了什么、为什么有效
2. **模式名称**：...

## 优势
- ...

## 劣势 / 局限
- ...

## 适用边界
- 适合：...
- 不适合：...

## 我们能从中学到什么
- 直接可复用的：（代码 / 架构 / 配置）
- 思路可借鉴的：（设计模式 / 接口约定）
- 需要避免的：（它的哪些做法不适合我们）
```

## 模式提取汇总表

分析完 Top 3 后，提取跨方案的共性模式。

```markdown
# 模式提取汇总

## 共识模式（大家都这么做）

| 模式 | 描述 | 出现在哪些方案中 | 我们的行动 |
|------|------|----------------|-----------|
| ... | ... | 方案A, 方案B, 方案C | 采纳 |

## 争议点（不同方案选择不同）

| 争议点 | 方案A 做法 | 方案B 做法 | 分析 | 我们的倾向 |
|--------|-----------|-----------|------|-----------|
| ... | ... | ... | 各有优劣，取决于... | 倾向 A，因为... |

## 反模式（踩过的坑）

| 反模式 | 描述 | 来源 | 我们如何避免 |
|--------|------|------|-------------|
| ... | ... | 方案A 的 Issue #123 | ... |

## 技术雷达定位

按 ThoughtWorks Technology Radar 的四环分类：

| 技术/方案 | 环位 | 理由 |
|----------|------|------|
| ... | Adopt（采纳）| 业界成熟，我们场景明确适用 |
| ... | Trial（试用）| 有潜力但需验证 |
| ... | Assess（评估）| 值得关注但暂不引入 |
| ... | Hold（观望）| 风险高或不适合当前阶段 |
```

## 开源项目健康度评估

对考虑 Fork 或深度依赖的开源项目，做健康度评估。

```markdown
# 开源健康度：[项目名]

## 维护活跃度（权重 30%）
- [ ] 最近 3 个月有 commit
- [ ] 最近 6 个月有 release
- [ ] Issue 平均响应时间 < 7 天
- [ ] PR 平均合并时间 < 14 天
- 评分：___ / 5

## 社区规模（权重 20%）
- [ ] Stars > 500
- [ ] Contributors > 10
- [ ] Fork > 50
- [ ] 有活跃的 Discussion / Discord / Slack
- 评分：___ / 5

## 代码质量（权重 20%）
- [ ] 有 CI/CD 管线
- [ ] 有测试覆盖
- [ ] 代码结构清晰，可读性好
- [ ] 依赖合理，无过度依赖
- 评分：___ / 5

## 文档完整度（权重 15%）
- [ ] README 清晰说明安装/使用
- [ ] 有 API 文档或使用指南
- [ ] 有 CHANGELOG
- [ ] 有 CONTRIBUTING 指南
- 评分：___ / 5

## 许可证与安全（权重 15%）
- [ ] 许可证与我们的场景兼容
- [ ] 无已知严重安全漏洞（CVE）
- [ ] 依赖链干净
- [ ] 有安全策略（SECURITY.md）
- 评分：___ / 5

## 综合评分
加权总分：___ / 5.0

| 评分区间 | 结论 |
|---------|------|
| 4.0-5.0 | 可以放心依赖 |
| 3.0-3.9 | 可用但需关注风险 |
| 2.0-2.9 | 谨慎使用，准备 Plan B |
| < 2.0 | 不建议依赖 |
```
