# 控制论 + Tutor 架构 — 框架详解

## 理论根节点

**目标即视角（Goals are Perspectives）**
- 目标不是"要做的事"，而是"看世界的镜头"
- 改变行为的本质是改变视角 → 改变身份
- 身份 = 固化的视角 + 自我保护机制

**控制论循环**：目标 → 行动 → 感知 → 比较 → 调整 → 重复
**自然界案例**：船只纠偏、恒温器、胰腺

---

## 改变的 3 阶段

1. **失调（Dissonance）**：感觉不属于当前生活，对缺乏进展极度厌倦
2. **不确定（Uncertainty）**：不知道下一步，要么实验要么迷失
3. **发现（Discovery）**：发现真正想追求的，6 个月做到 6 年的进步

> "你的人生最美好的时期总是出现在对缺乏进展感到极度厌倦之后。"

**关键洞见**：不能跳过阶段。Phase 1 制造失调，Phase 1.5 导航不确定，Phase 2 在发现后设计系统。

## 完整挖掘协议（深度模式 14 问）

### 揭示隐藏不满（4 问）
1. 你学会忍受的那种沉闷而持久的不满是什么？
2. 你反复抱怨但从未真正改变的是什么？写下过去一年最常表达的三个抱怨
3. 对于每个抱怨：如果有人观察你的行为（而非言语），他们会得出你真正想要什么？
4. 关于当前生活的什么真相，承认给你深深尊敬的人会让你无法忍受？

### 构建反愿景（7 问）
1. 如果未来 5 年什么都不变，描述一个普通周二的完整一天（从醒来到入睡）
2. 描述 10 年后：你错过了什么？什么机会关闭了？谁对你放弃了？
3. 你在生命尽头。你过了安全版本的生活。代价是什么？
4. 你生活中谁已经在过你描述的未来？想到成为他们感觉如何？
5. 要真正改变，你必须放弃什么身份？社交代价是什么？
6. 你没有改变的最令人尴尬的原因是什么？
7. 如果你的行为是自我保护，你在保护什么？这个保护的代价是什么？

### 创建最小可行愿景（3 问）
1. 忘记可行性。3 年后理想的周二是什么样的？（同等细节水平）
2. 让那种生活感觉自然需要相信自己什么？写身份陈述："我是那种...的人"
3. 如果你已经是那个人，这周你会做什么一件事？

## 身份抵抗详解

### 身份形成 8 步循环

```
1. 设定目标 → 2. 通过目标镜头感知 → 3. 注意"重要"信息
→ 4. 行动并接收反馈 → 5. 重复至自动化
→ 6. 行为成为身份 → 7. 捍卫身份 ⚠️ → 8. 身份塑造新目标
```

打破点：第 6-7 步。意识到"我是...的人"是可修改的代码。

### 为什么身份会抵抗

- 身份威胁触发与身体威胁**相同的**战斗/逃跑反应
- 这是进化机制，不是你的"软弱"
- 越深层的身份改变，抵抗越强烈

### 童年编程（Identity Origin）

大多数人的核心身份不是自己选择的，而是**继承**的：
- 你的生存目标让你依赖父母
- 父母通过奖惩教你"正确的"信念和价值观
- 除非你采用他们的标准，否则会被惩罚
- 你的父母也经历了同样的过程（工业时代的成功观念被代代相传）

**关键问题**："你现在相信的关于成功/安全/价值的东西，有多少是你自己验证过的，有多少是从小被告知的？"

这就是为什么 Phase 1 需要问"你的旧身份从哪来"——只有看到身份的来源，才能意识到它是可修改的。

### 抵抗的 5 种伪装

| 伪装 | 真实含义 | 识别方法 |
|------|---------|---------|
| "我太忙了" | 旧身份在争夺时间 | 你有时间刷手机吗？ |
| "我不够好" | 旧身份在制造自我怀疑 | 在开始之前你怎么知道？ |
| "再准备准备" | 旧身份在制造安全幻觉 | 什么时候才算"准备好"？ |
| "这不适合我" | 旧身份在自我保护 | 你试过多久了？超过 2 周吗？ |
| "大家都觉得我疯了" | 群体身份压力 | 你想活成"大家"的样子吗？ |

### 应对框架

**紧急应对**（出现抵抗信号时）：
1. 命名："这是身份抵抗，不是真正的问题"
2. 暂停：24 小时内不做任何"放弃"的决定
3. 回到 WHY：打开系统文件，重读反愿景 + 愿景 + 身份陈述

**系统应对**（在系统中内置）：
1. 在系统文件中写一段"给想放弃的自己的信"
2. 每月重新激活反愿景（如果无感，重写）
3. 找一个人作为"身份锚"——在你动摇时提醒你

## 阶段适配设计指南

### 9 阶段心智发展模型

| 阶段 | 核心 | 案例 |
|------|------|------|
| 1-3 | 外部导向 | 从冲动到顺从到群体认同 |
| 4-5 | 转折期 | 发现内在生活，建立自己原则 |
| 6-7 | 元认知 | 看到原则的局限，系统思维 |
| 8-9 | 超越 | 框架皆虚构，自我消融 |

### 阶段定位问题

1. "你的目标主要来自你自己，还是来自社会/家庭的期望？"
   - 社会/家庭 → 可能 3-4 阶段
   - 自己，但很确定是"正确的" → 可能 5 阶段
   - 自己，但知道可能会变 → 可能 6-7 阶段

2. "你如何看待'失败'？"
   - 证明我不行 → 4-5 阶段
   - 系统反馈 → 6-7 阶段
   - 没有所谓失败 → 8+ 阶段

3. "你能同时持有两个矛盾的观点吗？"
   - 不能，必须选一个 → 4-5 阶段
   - 能，但会选择一个 → 6-7 阶段
   - 能，且享受矛盾 → 8+ 阶段

### 系统适配矩阵

**4-5 阶段用户的系统**：
- GOAL：具体、可量化的外在目标 + 身份探索
- ACT：详细的知识树 + 明确的学习路径 + 频繁的自检
- SENSE：每日打卡 + 量化指标 + 6 个觉察打断
- COMPARE：每日 + 每周严格执行
- ADJUST：遵循规则表，一次改一个变量

**6-7 阶段用户的系统**：
- GOAL：以身份陈述为核心，外在目标作为实验
- ACT：宽松的知识框架 + 自主探索 + 项目驱动
- SENSE：能量追踪 + 情感雷达 + 减少打卡
- COMPARE：周复盘为主，日复盘可选
- ADJUST：基于直觉 + 数据的混合决策

**8+ 阶段用户的系统**：
- GOAL：方向感而非具体目标
- ACT：跟随好奇心，最小结构
- SENSE：纯觉察，不需要打卡
- COMPARE：月度反思为主
- ADJUST：信任过程，顺应自然

## Tutor 架构在 ACT 模块中的应用

### 知识架构化（通用模式）

**1. 领域分解**：任何目标 → 6-10 个加权知识区域
- 权重分配：生存/基础 > 核心技能 > 进阶能力

**2. 学习路径**：按依赖关系分 3-4 个 Phase

**3. 苏格拉底自检**：
- 能解释 = 理解 | 能变通 = 掌握 | 能教别人 = 精通

**4. 行动日志**（Tutor 会话记录协议的应用）：
- 每次实践 → 记录做了什么/学了什么/卡在哪/什么感觉/下一步
- 这些记录是 SENSE 模块的核心输入

### 不同目标类型的 ACT 指引

| 类型 | 知识树特征 | 验证方式 |
|------|-----------|---------|
| 学习类 | 按技术依赖排序 | 代码能跑 + 能独立解决问题 |
| 技能类 | 理论+实操并重 | 模拟验证 + 风险控制 |
| 身体类 | 营养/训练/恢复/心理 | 身体指标 + 主观感受 |
| 创作类 | 内容/分发/变现/社区 | 发布频率 + 受众反馈 |
| 职业类 | 技术/项目/面试/人脉 | 能通过面试 + 有展示项目 |

## 游戏化框架

6 个组成部分如同心圆力场：

反愿景（紧迫感）→ 愿景（方向感）→ 1 年目标（优先级）→ 1 月项目（成就感）→ 每日杠杆（进度感）→ 约束（创造力边界）

---

## 自然度阶梯

改变内化的 5 个阶段：

```
强迫 → 习惯 → 偏好 → 身份 → 本能
 ↑                              ↑
需要意志力                 无法想象其他方式
```

**真正的改变标志**：不是"我坚持做到了"，而是"我无法想象不这样做"。
