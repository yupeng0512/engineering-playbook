# Skill Taxonomy - Skill 类型分类体系

> 11 种 Skill 类型，帮助快速定位方法论和设计模式

---

## 为什么需要分类？

不同类型的 Skill 适用不同的方法论和工作流程。正确识别 Skill 类型能帮助：
1. **快速定位**：找到该类型的专家和最佳实践
2. **设计模式**：应用经过验证的 Skill 结构
3. **质量标准**：明确该类型 Skill 的成功标准

---

## 11 种 Skill 类型

### 1. Generation（生成类）

**定义**：从零创建内容或文档

**触发词**：
- "帮我写..."
- "生成一个..."
- "创建一份..."

**典型场景**：
- 写 PRD（产品需求文档）
- 写周报/月报
- 生成测试用例
- 写招聘 JD
- 写技术文档

**核心方法论**：
- Barbara Minto（金字塔原理）
- Amazon 6-Pager
- Marty Cagan（PRD 框架）

**输出特征**：
- 结构化文档
- 明确的模板
- 完整的内容

**质量标准**：
- 逻辑清晰（结论先行）
- 结构完整（无遗漏）
- 易于阅读（格式规范）

---

### 2. Insight / Summary（洞察和总结类）

**定义**：理解复杂信息，提炼关键洞察

**触发词**：
- "帮我理解..."
- "总结一下..."
- "提取关键点..."

**典型场景**：
- 理解代码库
- 总结会议纪要
- 提炼文章要点
- 分析竞品

**核心方法论**：
- Zettelkasten（卡片盒笔记）
- Feynman Technique（费曼学习法）
- Pyramid Principle（金字塔原理）

**输出特征**：
- 层级化结构
- 关键洞察突出
- 简洁明了

**质量标准**：
- 抓住核心（不是简单摘抄）
- 有洞察（提炼规律）
- 易于理解（降维呈现）

---

### 3. Decision（决策类）

**定义**：帮助做出选择或决策

**触发词**：
- "帮我决定..."
- "应该选择..."
- "评估优劣..."

**典型场景**：
- 技术选型
- 招聘决策
- 战略选择
- 优先级排序

**核心方法论**：
- Jeff Bezos（Type 1/2 Decisions）
- Charlie Munger（心智模型）
- Weighted Decision Matrix（加权决策矩阵）

**输出特征**：
- 清晰的选项对比
- 决策标准
- 推荐方案 + 理由

**质量标准**：
- 标准明确（可量化）
- 权衡清晰（利弊分析）
- 可行性强（可执行）

---

### 4. Evaluation（评估类）

**定义**：评估质量、风险或价值

**触发词**：
- "帮我评估..."
- "审查一下..."
- "评审..."

**典型场景**：
- 架构评审
- 代码审查
- 方案评审
- 投资评估

**核心方法论**：
- ATAM（架构评审）
- AWS Well-Architected Framework
- Google SRE（可靠性评估）
- Code Review Best Practices

**输出特征**：
- 多维度评估
- 打分或等级
- 改进建议

**质量标准**：
- 维度完整（不遗漏）
- 标准明确（可量化）
- 建设性强（有具体建议）

---

### 5. Diagnosis（诊断类）

**定义**：找出问题根因，提供解决方案

**触发词**：
- "为什么..."
- "帮我找出问题..."
- "诊断..."

**典型场景**：
- 性能问题诊断
- 故障排查
- 用户流失分析
- 增长瓶颈诊断

**核心方法论**：
- 5 Whys（丰田生产方式）
- Fishbone Diagram（鱼骨图）
- Google SRE Troubleshooting
- First Principles（第一性原理）

**输出特征**：
- 问题树（层层追溯）
- 根因分析
- 解决方案

**质量标准**：
- 追根溯源（不只是表象）
- 逻辑严密（因果链清晰）
- 可验证（能测试假设）

---

### 6. Persuasion（说服类）

**定义**：说服他人接受观点或方案

**触发词**：
- "帮我说服..."
- "写一个提案..."
- "如何推销..."

**典型场景**：
- 项目提案
- 销售话术
- 投资人演讲
- 内部推广

**核心方法论**：
- SPIN Selling（Neil Rackham）
- Pyramid Principle（结论先行）
- Storytelling（故事化表达）
- Ethos-Pathos-Logos（亚里士多德修辞学）

**输出特征**：
- 结论先行
- 强调价值
- 预判反对意见

**质量标准**：
- 针对性强（了解听众）
- 逻辑清晰（易于接受）
- 行动导向（明确下一步）

---

### 7. Planning（规划类）

**定义**：制定计划和路线图

**触发词**：
- "帮我规划..."
- "制定计划..."
- "怎么安排..."

**典型场景**：
- Sprint 规划
- 项目计划
- 职业发展规划
- 产品路线图

**核心方法论**：
- Scrum Planning
- OKR（目标和关键结果）
- Gantt Chart（甘特图）
- Agile Estimation（敏捷估算）

**输出特征**：
- 时间线
- 里程碑
- 依赖关系
- 资源分配

**质量标准**：
- 目标明确（SMART）
- 可执行（分解到位）
- 可追踪（有检查点）

---

### 8. Optimization（优化类）

**定义**：改进现有方案或流程

**触发词**：
- "如何优化..."
- "提升..."
- "改进..."

**典型场景**：
- 性能优化
- 流程优化
- 成本优化
- 用户体验优化

**核心方法论**：
- Lean Manufacturing（精益生产）
- 80/20 Rule（帕累托法则）
- A/B Testing
- Profiling and Benchmarking

**输出特征**：
- 现状分析
- 瓶颈识别
- 优化方案
- 预期收益

**质量标准**：
- 数据驱动（有基准线）
- 聚焦瓶颈（不平均用力）
- 可衡量（有指标）

---

### 9. Learning / Tutorial（学习和教学类）

**定义**：帮助用户学习或教授知识

**触发词**：
- "教我..."
- "如何学习..."
- "讲解一下..."

**典型场景**：
- 技术学习
- 新人培训
- 知识传承
- 技能提升

**核心方法论**：
- Feynman Technique（费曼学习法）
- Spaced Repetition（间隔重复）
- Bloom's Taxonomy（布鲁姆教育目标分类）
- Socratic Method（苏格拉底式提问）

**输出特征**：
- 渐进式讲解
- 案例和练习
- 知识检查点

**质量标准**：
- 深入浅出（易于理解）
- 循序渐进（符合认知规律）
- 实践导向（能应用）

---

### 10. Automation（自动化类）

**定义**：自动化重复任务

**触发词**：
- "自动化..."
- "批量处理..."
- "一键..."

**典型场景**：
- CI/CD 流程
- 数据处理
- 报告生成
- 代码格式化

**核心方法论**：
- DevOps Best Practices
- Infrastructure as Code（基础设施即代码）
- Scripting Patterns
- Idempotency（幂等性）

**输出特征**：
- 可执行脚本
- 错误处理
- 日志记录

**质量标准**：
- 可靠性（幂等、容错）
- 可维护（清晰文档）
- 可扩展（参数化）

---

### 11. Coordination（协调类）

**定义**：协调多方资源或任务

**触发词**：
- "协调..."
- "安排..."
- "组织..."

**典型场景**：
- 跨团队协作
- 会议安排
- 资源调配
- 项目管理

**核心方法论**：
- Scrum Master 实践
- RACI Matrix（责任分配矩阵）
- Stakeholder Management
- Communication Plan

**输出特征**：
- 任务分配
- 时间表
- 沟通计划

**质量标准**：
- 职责明确（无重叠和遗漏）
- 时间合理（考虑依赖）
- 沟通到位（信息同步）

---

## 复合类型

很多 Skill 是复合类型，例如：

### 架构评审 = Evaluation + Diagnosis
- Evaluation：评估架构质量
- Diagnosis：诊断潜在风险

### PRD 写作 = Generation + Planning
- Generation：生成文档
- Planning：规划产品路线

### 技术选型 = Decision + Evaluation
- Decision：做出选择
- Evaluation：评估各选项

## 如何使用分类

### Step 1: 识别主类型

根据用户需求，判断主要类型：

```
用户："帮我写一个 PRD"
→ 主类型：Generation
→ 次类型：Planning

用户："评审这个架构设计"
→ 主类型：Evaluation
→ 次类型：Diagnosis
```

### Step 2: 查找方法论

根据类型，查找 `methodology-database.md` 中的相关专家：

```
Generation → Barbara Minto, Amazon 6-Pager
Evaluation → ATAM, AWS Well-Architected
Decision → Jeff Bezos, Charlie Munger
```

### Step 3: 应用设计模式

每种类型有典型的 Skill 结构：

#### Generation 类 Skill 结构
```markdown
## 你的角色
你是 [领域] 写作专家

## 输入要求
- 背景信息
- 目标受众
- 约束条件

## 工作流程
1. 理解背景和目标
2. 构建大纲
3. 填充内容
4. 审查和优化

## 输出模板
[详细的文档模板]

## 黄金案例
[优秀示例]

## 反模式
[常见错误]
```

#### Evaluation 类 Skill 结构
```markdown
## 你的角色
你是 [领域] 评审专家

## 评审维度
1. 维度 1
2. 维度 2
3. 维度 3

## 评分标准
[具体的判断标准]

## 工作流程
1. 理解背景
2. 多维度评审
3. 识别风险
4. 提供建议

## 输出格式
[评审报告模板]
```

| Evaluation | 维度完整、标准明确、建设性 |
| Diagnosis | 追根溯源、逻辑严密、可验证 |
| Persuasion | 针对性强、逻辑清晰、行动导向 |
| Planning | 目标明确、可执行、可追踪 |
| Optimization | 数据驱动、聚焦瓶颈、可衡量 |
| Learning | 深入浅出、循序渐进、实践导向 |
| Automation | 可靠性、可维护、可扩展 |
| Coordination | 职责明确、时间合理、沟通到位 |
