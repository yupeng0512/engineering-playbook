## 判断力卡片：AI Agent 架构选型与设计

> 来源: ai-agent-learning/notes/agent-architecture-design.md
> 锻造日期: 2026-02-25
> 领域: 技术架构

### 核心判断
在需要生产级 AI Agent 的场景下，应先用「实习生标准」四维框架（执行/思考/进化/工程保障）做任务分析，再按任务复杂度、依赖关系和迭代需求选择 ReAct / Plan-Execute / Reflexion 的组合，因为任务特征与架构能力存在可归纳的映射关系，选型错误是失败的主要来源。

### 关键问题
**这个任务的核心约束是什么？**（成本 / 质量 / latency / 可解释性 / 可扩展性）

### 适用边界
- **成立条件**: 任务边界相对清晰；LLM 能力与当前假设基本一致；有可观测的失败模式可迭代
- **失效条件**: 任务高度开放或边界模糊；模型能力快速跃迁；成本不是主要约束却过度优化
- **不确定区域**: 多模态、长链推理、强实时交互等新兴场景的架构适配

### 心智模型
- **系统思维**：把架构选型放在「目标 → 约束 → 方案」链条中，而不是孤立选型
- **逆向思维**：先问「什么情况下这套方案会失败」，再反推成立前提和失效条件

### 反面论证
反对理由：该框架依赖对任务复杂度的主观判断，且「实习生标准」未必适用于所有领域。仍采用该判断的原因：复杂度判断可通过步骤数、依赖关系、迭代需求等显式维度收敛，而「实习生标准」提供可操作的验收基线，比抽象原则更利于落地和迭代。
