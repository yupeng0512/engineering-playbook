## 判断力卡片：Agent 工具设计的投入回报远超提示工程

> 来源: Anthropic "Writing Effective Tools for Agents" + Building Effective Agents
> 锻造日期: 2026-02-25
> 领域: 技术架构

### 核心判断
在开发 AI Agent 时，应将至少 50% 的优化时间投入工具接口设计（ACI），而非提示工程——因为工具格式对 LLM 使用准确率的影响远超直觉（Diff 格式改为完整文件，错误率 -87.5%）；工具是确定性系统与非确定性 Agent 之间的契约，设计质量直接决定 Agent 的上限。用 Claude 优化 Claude 自己的工具（元循环自举）是投入产出比最高的优化路径。

### 关键问题
**如果一个实习生只看你的工具文档，能否零歧义地正确使用这个工具？**（Agent 的理解能力约等于一个仔细但字面化理解的实习生）

### 适用边界
- **成立条件**: Agent 需要调用外部工具完成任务；工具有明确的输入/输出规范；可以通过评估（eval）衡量工具使用准确率
- **失效条件**: 纯对话场景无工具调用；工具极简（只有 1-2 个参数，难出错）；模型本身能力不足以理解任何格式的工具
- **不确定区域**: 工具数量极多（50+）时，工具选择本身是否成为瓶颈而非工具设计；未来模型是否能自动适配低质量工具文档

### 心智模型
- **逆向思维**: 不问"如何写出更好的提示"，而问"如何让工具更难用错"（防呆 > 防错）
- **激励结构**: Anthropic 在 SWE-bench 竞赛中投入 50% 时间优化工具，这是经过 ROI 验证的资源分配，非理论推导

### 反面论证
反对观点：提示工程的灵活性更高、迭代更快，工具一旦定型修改成本大；在快速原型阶段不应过早优化工具。仍坚持本判断的原因：工具设计错误的代价远大于提示错误——坏的工具接口导致 Agent 系统性失败，而坏的提示通常只影响局部质量；防呆设计（如强制绝对路径、禁止 diff 格式）一旦实施，永久性地消除整类错误。
