## 判断力卡片：Skill 注入的倒 U 型增益法则——精准挂载胜于全面覆盖

> 来源: SkillsBench: Benchmarking How Well Agent Skills Work Across Diverse Tasks (BenchFlow + Amazon/ByteDance/Foxconn + Stanford/CMU/UC Berkeley/Columbia/Oxford)
> 锻造日期: 2026-02-27
> 领域: 技术架构

### 核心判断
向 AI Agent 注入外部程序性知识（Skills/SOP）时，增益遵循倒 U 型曲线：**收益 = f(领域稀缺度 × 结构精简度) — 超过阈值则产生负增益**。应将 Skill 库严格限制在 2-3 个模块/任务，采用 Compact/Detailed 结构（而非 Comprehensive），并优先为模型知识稀缺的垂直领域编写 Skill——因为 7,308 次执行轨迹的实证数据表明，过度注入会引发冲突指导和认知过载，导致性能倒退。

### 关键问题
**模型在这个领域的先验知识有多强？如果已经很强（如通用编程），注入 Skill 可能弊大于利。**

（先验充分 → 裸跑或极简提示；先验稀缺 → 精准挂载 2-3 个 Compact 型 Skill；不确定 → A/B 测试量化 Delta）

### 适用边界
- **成立条件**: Agent 执行垂直领域任务（医疗/制造/金融合规等模型先验稀缺领域）；Skill 文档 ≤ 3 个模块、中位数 ~2.3KB；上下文窗口有限（8K-128K 实际可用区间）；有 Pass/Fail 可量化的验收标准
- **失效条件**: 单轮简单对话（无需 Skill）；模型对任务已有极强先验（通用编程、数学）；未来模型窗口和注意力机制突破使上下文噪音不再是问题
- **不确定区域**: 倒 U 型曲线的精确拐点是否因模型家族而异（Claude 系 vs Gemini 系表现差异显著）；Harness 的调度实现（隐式扫描 vs 显式调用）对增益的影响权重；"模型傲慢"现象（知道 Skill 但不用）的根因是否可通过框架层面解决

| 指标 | 数值 | 来源 |

### 心智模型
- **倒 U 型曲线（Yerkes-Dodson 法则）**: 刺激（信息注入）存在最优区间，过少无效，过多致损——与人类认知负荷理论同构
- **奥卡姆剃刀**: 技能文档的每一行都必须直接服务于任务通过率，否则就是噪音
- **瓶颈理论**: 上下文窗口是 Agent 系统的核心瓶颈，Skill 占据的 Token 与模型"思考空间"直接竞争
- **概率思维**: 增益的领域异质性极大（4.5pp ~ 51.9pp），脱离领域语境讨论"Skill 有没有用"毫无意义

### 反面论证
反对观点：随着上下文窗口扩大（Gemini 2M+）和注意力机制改进，"精简"策略可能变得不必要；全面型文档在极长窗口下可能反而有优势。仍坚持本判断的原因：SkillsBench 的实证数据在当前主流窗口下已清晰证明倒 U 型关系；即使窗口扩大，信噪比优化的边际收益不会为零——Gemini 3 Pro 接入 Skill 后 Token 消耗反降 6%，说明精准知识在任何窗口大小下都能缩短搜索空间；此外，"自生成 Skill 失败"揭示的 Unknown Unknowns 问题是模型能力的根本局限，不随窗口扩大而消失。

### 验证状态
论文实证已验证（7,308 次执行轨迹，84 任务，11 领域）。待个人实践验证：在自有 Skill 库中执行 2-3 模块约束，量化精简前后的通过率 Delta。
