## 判断力卡片：AI Agent 架构选型——简单性优先

> 来源: Anthropic 构建有效 AI Agent 的实践指南 (Erik Schluntz & Barry Zhang)
> 锻造日期: 2026-02-25
> 领域: 技术架构

### 核心判断
在构建 AI 驱动的任务系统时，应默认从单次 LLM 调用或 Workflow 开始，只在任务步骤不可预测且复杂度能带来可衡量的性能提升时升级为 Agent；同时，将至少与提示工程同等的精力投入工具接口（ACI）设计，因为工具格式和文档对 LLM 使用准确率的影响远超直觉。

### 关键问题
**这个任务的步骤是可预测的吗？如果不可预测，复杂度提升能否被量化证明？**

（若步骤可预测 → Workflow；若不可预测且无法量化收益 → 暂不升级；若不可预测且能证明 ROI → Agent）

### 适用边界
- **成立条件**: 任务有明确成功/失败标准、成本与准确率可测量、团队能理解底层 LLM API、存在工具调用场景
- **失效条件**: 任务本质开放无终点、业务强制要求用 Agent、评估无法量化、工具不可控
- **不确定区域**: LLM 能力跃迁后「70% 用 Level 0」是否仍成立；垂直领域合规要求；多模态/具身 Agent 是否适用同一框架

### 心智模型
- **奥卡姆剃刀**: 如无必要，勿增实体——复杂度必须证明其价值
- **激励结构分析**: Anthropic 作为 Claude 制造商，其利益与用户成功基本一致（成功案例 → 更多订阅），但也存在推广自家工具的倾向
- **概率思维**: 置信度随任务可预测性与可测量性变化

### 反面论证
反对理由：Agent 代表前沿，过早简化可能限制创新潜力；某些场景下「先跑起来再优化」比「先简单再升级」更高效。仍选择此判断的原因：量化证据可复现（SWE-bench 数据）；简单优先是风险控制原则；可测量、可调试原则长期有效。

### 新增实证：SkillsBench "精简为王"数据（2026-02-27）

SkillsBench（7,308 次执行轨迹）从 Skill 设计维度为"简单性优先"提供了量化支撑：

| 证据 | 数据 | 与本卡片的关系 |
|------|------|---------------|
| Compact > Comprehensive | Compact +17.1pp vs Comprehensive -2.9pp | "如无必要勿增实体"的直接量化验证 |
| 自生成 Skill 失败 | 比盲考基线还低 1.3pp | 过度工程化（让模型先写 SOP 再执行）的反面案例 |
| 最优杠杆案例 | openpyxl 透视表 API 规范 → 6 模型从 0% 跃至 80%+（+85.7pp） | 精确、最小化的知识注入比全面文档有效得多 |

**核心启示**：不仅架构选型要简单优先，给 Agent 喂的知识包也要简单优先。一个精准的 API 速查表（~2.3KB）的增益，远超一份试图涵盖所有边缘情况的完整文档。

### 验证状态
待个人实践验证。SkillsBench 论文（2026-02-27 补充）提供了 84 任务 × 7 模型配置的大规模实证支持"精简为王"原则。建议在下一个项目中记录架构选型依据、对比工具优化前后准确率，并在 6 个月后回看「70% Level 0」在本团队的实际占比。
